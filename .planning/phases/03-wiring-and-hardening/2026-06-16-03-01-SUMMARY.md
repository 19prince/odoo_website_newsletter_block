## Decisions Made

- `.include()` over `.extend()` for JS widget: extend creates a new unregistered class not picked up by existing event bindings; include patches prototype in-place
- No `super().subscribe()` call: recaptcha token verification pops from request.params as a side-effect — calling it twice would corrupt the flow
- `address_name` as flat RPC param (not nested): controller receives it via `**post` kwargs without any special handling
- list_id=0 guard returns structured toast response instead of raising exception — user-facing error, not a server error
