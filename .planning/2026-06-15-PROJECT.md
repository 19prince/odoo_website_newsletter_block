# odoo_website_newsletter_block

## What This Is

An Odoo website builder module that provides a styled, reusable newsletter subscription snippet. Editors drag the block onto any page or blog post, configure the mailing list once, and visitors submit their name and email to subscribe. Designed to be client-deployable with generic defaults and compatible with Odoo 18 and 19.

## Core Value

A newsletter CTA that editors drop in once and never have to restyle — fully editable in the builder, always on-brand.

### Active

- [ ] Snippet appears in website builder sidebar under "Contact & Forms" and is draggable onto any page or blog post
- [ ] Block has editable headline, body text, and sub-caption (inline `oe_editable`)
- [ ] Block background is configurable via Odoo's standard `o_cc` color palette (no custom color picker)
- [ ] Inline form row: name field, email field, submit button — single-column stacked below headline text
- [ ] Mailing list is configurable per block instance via snippet options panel (inherits `website_mass_mailing` `data-list-id` pattern)
- [ ] Form submits via POST to custom controller `/website/newsletter/subscribe`
- [ ] Controller creates or matches a `mailing.contact` record and adds to the configured `mailing.list`
- [ ] Success state is an inline swap (form hidden, thank-you message shown) — no page reload or redirect
- [ ] Module is compatible with Odoo 18 and Odoo 19 (no version-specific internals)
- [ ] Generic defaults throughout — no 19 Prince branding baked in

### Out of Scope

- Custom hex color picker — using `o_cc` palette for version stability
- Image background support — not needed for newsletter CTA use case
- Redirect-based thank-you pages — inline swap is simpler and keeps the reader on page
- Custom subscriber model — `mailing.contact` reuses proven email marketing infrastructure
- Split first/last name fields — single "name" field matches reference design and maps cleanly to `mailing.contact.name`
- Double opt-in flow — handled by Odoo's existing mailing list settings, not this module

## Context

- Odoo local dev environment at `~/Code/odoo18` (Odoo 18, `brain_dev` db, admin/admin)
- Existing `website_mass_mailing` module provides `s_newsletter_block` (email-only) and the `data-list-id` snippet options pattern this module inherits
- Reference design: BeehiV-style inline subscription card (headline + body + inline form row + sub-caption)
- 19 Prince is an Odoo implementation consultancy — this module will be used on client sites
- The website editor JS API has shifted between major Odoo versions; all builder interaction uses QWeb + `o_cc` + the stable `data-list-id` options mechanism to avoid version lock

## Constraints

- **Compatibility**: Odoo 18 + 19 — use only stable primitives (QWeb templates, `o_cc`, `data-list-id`); no version-specific editor JS APIs
- **Dependency**: `website_mass_mailing` — required for mailing list snippet options mechanism
- **Deployment**: Client-deployable — generic copy, no hardcoded list IDs or brand assets
