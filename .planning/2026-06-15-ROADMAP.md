# Roadmap: odoo_website_newsletter_block

## Overview

Three phases deliver a working, client-deployable newsletter subscription snippet for the Odoo 18 website builder. Phase 1 scaffolds a loadable module. Phase 2 builds the complete QWeb template with all editable zones, correct CSS classes, data attributes, and the inline success swap — giving editors a functional block to drop on any page. Phase 3 wires the mailing list picker, extends the name field, and hardens the list_id=0 guard so the block is safe to publish on any client site.

## Phases

- [x] **Phase 1: Scaffold** - Module installs cleanly with correct dependency and security access
- [ ] **Phase 2: Core Block** - Editors can drop, style, and edit a working newsletter block; visitors can subscribe
- [ ] **Phase 3: Wiring and Hardening** - Mailing list is configurable per block; name is captured; unconfig'd blocks are safe

## Phase Details

### Phase 1: Scaffold
**Goal**: Module loads on Odoo 18 without errors and is ready for template work
**Depends on**: Nothing (first phase)
**Requirements**: MODL-01, MODL-02, COMPAT-01
**Success Criteria** (what must be TRUE):
  1. `odoo/bin/odoo -u odoo_website_newsletter_block` completes with no ERROR or WARNING lines in the log
  2. Module appears in Apps list and can be installed/uninstalled cleanly
  3. `website_mass_mailing` is the only declared dependency in `__manifest__.py` (no `google_recaptcha` or `website` listed directly)
  4. Website designer role can read `mailing.list` records (security CSV grants access)

### Phase 2: Core Block
**Goal**: Editors can drag the newsletter block onto any page, edit all text zones, set background color, and visitors can fill in name + email and see the inline thank-you swap
**Depends on**: Phase 1
**Requirements**: SNIP-01, SNIP-02, SNIP-03, EDIT-01, EDIT-02, EDIT-03, EDIT-04, EDIT-05, LIST-02, FORM-01, FORM-04
**Success Criteria** (what must be TRUE):
  1. Newsletter block appears in the website builder sidebar under "Contact & Forms" and can be dragged onto a page or blog post
  2. Headline, body text, sub-caption, and thank-you message are each editable in-place without leaving the builder
  3. Background color changes when the editor selects a palette swatch from the standard color section picker
  4. Visitor can type a name and email address and submit the form; the form area is replaced by the thank-you message without a page reload
  5. Block survives a save-and-reload with form inputs intact (t-forbid-sanitize="form" in effect)
**Plans**: 1 plan

### Phase 3: Wiring and Hardening
**Goal**: Mailing list is selectable per block instance, submitted contacts are created with their name, and blocks with no configured list cannot corrupt the database
**Depends on**: Phase 2
**Requirements**: LIST-01, FORM-02, FORM-03, FORM-05, COMPAT-02
**Success Criteria** (what must be TRUE):
  1. Editor can open the snippet options panel and pick a mailing list; that list's ID is written to `data-list-id` on the `.s_newsletter_list` element
  2. After a visitor submits, a `mailing.contact` record exists in Odoo with the visitor's name populated in the `name` field
  3. Submitted visitor appears under the selected mailing list in Odoo's Email Marketing contacts
  4. Submitting a block with `data-list-id="0"` (no list configured) returns an error rather than a 500/FK violation
  5. Module passes end-to-end subscribe flow on the local Odoo 18 dev environment (brain_dev db)
